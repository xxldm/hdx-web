# Login Page Design Override

## 目标

- 登录页采用 `Liquid Glass` 风格，降低原登录框过于正式、规矩的观感。
- 登录页是整体用户端工具箱风格的第一屏，不是一次性页面特例；后续页面应延续轻松、年轻、个人化的气质。
- 保留 HDX 工具箱的可信感，不做企业工作台、营销页或复杂 onboarding。
- 不改变登录、Desktop Online 配置、CSRF、BFF 和 i18n 行为。

## 视觉规则

- 背景使用纯 CSS 明暗双态流体高光和玻璃模糊，不依赖单一背景图资产。
- 主登录容器使用高透明度玻璃材质，必须保证浅色文字和表单控件可读。
- 桌面端可以使用非对称双栏：左侧品牌氛围，右侧登录面板。
- 移动端只保留登录主路径，避免装饰元素挤占表单空间。
- 图标使用 Lucide，不使用 emoji。

## 交互规则

- Hover 只改变边框、阴影或背景透明度，不使用会造成布局位移的缩放。
- 动画仅用于背景光斑和玻璃高光，并遵守 `prefers-reduced-motion`。
- 登录错误和 Desktop Online 配置错误仍使用现有 `UAlert`。

## 禁止项

- 不用 `-Ob`、release、native build 或后端状态作为 UI 文案。
- 不新增外部图片、远程字体或第三方视觉依赖。
- 不把 Desktop Online 配置做成嵌套卡片；在登录面板内使用分隔区。
